Introduction

    The human mind now has an attention span of 8 seconds. We would like to retain that attention and keep the user engaged. So we decided to introduce noise cancellation.

Problem statement

    The usage of video conferencing has spiked, due to WFH conditions. This means there is a shift in work environment and there is a push to maintain the status quo. It becomes absolutely necessary to maintain good audio quality.

Solution Description

  • Video chat application with inbuilt noise suppression based on Jean-Marc Valin's work RNNoise
  • Hybrid approach as it tries to get DNN quality with DSP complexity
  • Embedded the RNNoise library using Emscripten and Webassembly
  • Edge based audio Processing
  • A Peer to Peer Web-Video Application which uses the Webrtc framework to interact with the user

DESIGN

RNNoise Design

  • The concept is basesd on traditional DSP approach
  • Divide the spectrum into 22 "critical bands"
  • RNN attenuates each of these bands
  • Uses a puitch filter to drop noise in between harmonics

Complexity based on 48 khz

  • Audio file of 48khz requires
    • 215 neurons with approx of 88k weights
    • Total complexity around 40 MFLOPS
  • Unvectorized C Code
    • 1.3% CPUS on traditional x86 Intel Arch
    • In real time with it running as a WASM module in the Browser
    • Slight hiccups in Mozilla Firefox

Conclusion

  • The noisy datasets which are available are of artificial quality.The performance is expected to improve over time as we obtain better datasets
  • Noise for everyone is different in different contexts
  • A combination of RNN with Digital Signal Processing techniques yield the quality of a DNN without the CPU throttling
